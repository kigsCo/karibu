# (Re)generating seed.sql

`supabase/seed.sql` mirrors the prototype's launch data. It runs automatically at the end of `supabase db reset` (after all migrations). It must be **idempotent-friendly and re-runnable** from a clean DB.

## When to use
The prototype constants in `src/KaribuApp.jsx` changed, or you are bootstrapping `seed.sql` for the first time.

## Source constants -> tables
All source data is in `src/KaribuApp.jsx` (read it before generating):

| Prototype constant (line) | Target table | Key field mapping |
|---|---|---|
| `cities` (~L20) | `cities` | `key`->`slug`, `label`->`name`, `tagline`, `hoods`->`hoods` (text[]); set `is_active = true` |
| `categories` (~L28) | `categories` | `key`->`slug`, `label`, `blurb`, `Icon` name (e.g. `Hotel`)->`icon` (text), array index->`sort_order` |
| `categories[].subTypes` | `sub_types` | parent via `category_id` subquery, `key`->`slug`, `label`, `Icon`->`icon`, index->`sort_order` |
| `recommended` (~L188) + `salonsList` (~L246) | `businesses` | `name`, `hood`, `rating`, `reviews`->`review_count`, `price`->`price_range`, `tags`->`tags`, `services`->`services_json`, `badge`->`tier`, contact fields; `status='active'` |
| `reviewsSample` (~L321) | `reviews` | `name`->`reviewer_name`, `country`->`reviewer_country`, `rating`, `text`->`body`; `status='published'`, `published_at=now()` |
| `guides` (~L358) | `guides` | `id`->`slug`, `category`, `cityKey`->`city_id` subquery, `title`, `subtitle`, `readTime`->`read_time`, `author`, `body`(array)->`body_json` (jsonb), `featured`, `summary`; `is_published=true` |

Notes on the mapping:
- `badge` -> `tier`: `"Karibu Recommended"` => `recommended`; `"Verified"` => `verified`; `null` => `free`.
- `businesses.slug` = the prototype `id` (e.g. `posh`, `talisman`) — these become the public slugs.
- `category_id` / `sub_type_id` / `city_id` come from **slug subqueries** (next section).
- Reviews in the prototype attach to the showcased salon (Posh Palace); seed them against `slug = 'posh'`.
- Body length: `reviews.body` has a `CHECK (length(body) >= 40)`; the sample reviews already exceed that — keep them intact.

## Golden rule: reference FKs by slug subquery, never hardcoded UUIDs
UUIDs are generated by `uuid_generate_v4()` at insert time, so you cannot hardcode them. Resolve every FK with a subquery on the natural key (`slug`):
```sql
-- categories first, then sub_types referencing the category by slug
INSERT INTO categories (slug, label, blurb, icon, sort_order, is_active) VALUES
  ('hotels', 'Hotels & Housing', 'Stays for every length', 'Hotel', 0, true),
  ('beauty', 'Health & Beauty',  'Salons, spas, fitness',  'Sparkle', 5, true);

INSERT INTO sub_types (category_id, slug, label, icon, sort_order) VALUES
  ((SELECT id FROM categories WHERE slug = 'beauty'), 'hair', 'Hair Salons', 'Scissors', 0),
  ((SELECT id FROM categories WHERE slug = 'beauty'), 'nails','Nail Salons', 'Sparkle',  1);

-- a business: category_id / sub_type_id / city_id all via slug subqueries
INSERT INTO businesses
  (slug, name, category_id, sub_type_id, city_id, hood, price_range, tags,
   services_json, tier, status, rating, review_count)
VALUES (
  'posh', 'Posh Palace Salon',
  (SELECT id FROM categories WHERE slug = 'beauty'),
  (SELECT id FROM sub_types  WHERE slug = 'nails'
     AND category_id = (SELECT id FROM categories WHERE slug = 'beauty')),
  (SELECT id FROM cities WHERE slug = 'nairobi'),
  'Westlands', 'KSh 1,500-6,000',
  ARRAY['Gel nails','Braids','Pedicure'],
  '[{"name":"Classic manicure","price":"KSh 1,500"},{"name":"Gel manicure with art","price":"KSh 3,500"}]'::jsonb,
  'recommended', 'active', 4.8, 412
);

-- a review attached by business slug
INSERT INTO reviews
  (business_id, reviewer_name, reviewer_country, rating, body, status, published_at)
VALUES (
  (SELECT id FROM businesses WHERE slug = 'posh'),
  'Sarah M.', 'Germany', 5,
  'Absolutely the best nail salon experience I have had in Nairobi. Booked via WhatsApp, confirmed in 2 minutes. Gel set still perfect after 3 weeks.',
  'published', now()
);
```

## Ordering (FK dependencies)
Insert in dependency order so subqueries resolve:
1. `cities`
2. `categories`
3. `sub_types`
4. `businesses`
5. `reviews`
6. `guides`

## Keep it runnable
- After editing, run `supabase db reset` — it must apply all migrations then `seed.sql` with **zero errors**.
- `rating` / `review_count` on `businesses` are normally trigger/cron-maintained, but for seed data set sensible values directly so the prototype's numbers show on a fresh DB (the nightly `calculate-rankings` job will recompute `ranking_score`).
- Strip emoji flags from `reviewer_country` (store `'Germany'`, `'United States'`, not the flag glyph). The prototype shows flags in the UI layer.
- Do not hardcode `created_at` for things that should be "now"; use `now()` or omit (defaults apply).

## Common mistakes
- Hardcoding UUIDs for FKs (they don't exist until insert time) — always use slug subqueries.
- Wrong insert order, so a subquery returns NULL and the FK insert fails.
- A `reviews.body` under 40 chars (violates the CHECK).
- Forgetting `is_active = true` on cities (RLS then hides them from the public read) or `status = 'active'` on businesses.
- Passing a JS array/object literal instead of `ARRAY[...]` / `'...'::jsonb`.
